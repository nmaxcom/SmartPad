Future Syntax Ideas

Think of this as a product pitch: we want SmartPad to feel like you are talking to your math out loud. The syntax stays lightweight, but the results feel alive and visual. Every example below shows the actual result after =>.

Modeling and Outcomes

These ideas make documents feel more like a story: you define inputs, ask questions, and get clear answers without turning your notes into a spreadsheet.

Named outputs with := (headline numbers)

Sometimes you want a big result without creating yet another variable. A named output is a clean, readable way to say "show me the answer" and keep the variable panel tidy.

Example: bill total without storing a new variable

base = $120
tax = 12%
total := base + tax on base
total => $134.4

Total shows up as a result, but doesn't add a new variable entry.

Example: highlight a key metric in a long note

monthly revenue = $52500
operating margin = 22%
profit := monthly revenue * operating margin
profit => $11550

Goal seek / solver

Let SmartPad solve the puzzle for you. You write the relationships, then ask for the missing value that makes a target come true.

Example: "What discount gets me to $60?"

price = $80
total = price - (price * discount)
solve discount so total = $60
discount => 25%

Example: "What radius gives me area 50?"

area = PI * radius^2
solve radius so area = 50
radius => 3.99

Auto-variables (prefix shorthand)

Prefix variables (like $rate) so they pop instantly in a mixed paragraph. It feels like annotating your thoughts, not coding.

Example: quick parameters in a dense paragraph

$rate = 0.08
subtotal = $73.75
total = subtotal + subtotal * $rate => $79.65

Example: distinguish plan knobs from facts

$fee = $4
rides = 12
monthly total = rides * $fee => $48

Scenario and Time

This makes your document feel like a living model. You can switch between scenarios, scrub through time, and see how the same logic behaves under different conditions.

Scenario blocks

Scenario blocks override values only inside their lane. Compare plans without copying formulas or rebuilding your model.

Example: base vs promo pricing

price = $100
discount = 20%
total = discount off price => $80

@scenario Promo {
  discount = 30%
}
total => $70

Example: compare shipping options

items = 6
shipping = $4
total shipping = items * shipping => $24

@scenario Express {
  shipping = $8
}
total shipping => $48

Recurrence (monthly, yearly)

Recurrence syntax makes recurring values feel native. You define them once and they drive both calculations and timeline views.

Example: monthly rent to yearly total

rent monthly = $1250
yearly rent = rent * 12 => $15000

Example: weekly sessions to monthly count

sessions weekly = 3
sessions per month = sessions * 4 => 12

Time series ranges

Range syntax creates a series automatically. Any expression using that series becomes a chart-ready result without extra setup.

Example: 12-month forecast

t = 1..12
forecast = 1200 * 1.05^t
forecast => chart line

Example: savings ladder

month = 1..6
saved = 200 * month
saved => chart bar

Quality and Safety

When the math matters, the document should help you catch mistakes. These additions make assumptions visible and surface risk, not just results.

Assertions with pass/fail badges

Assertions behave like tests. When they fail, the line highlights and can notify you. It turns your model into something you can trust over time.

Example: budget guardrail

total = $920
total < $1000 ! => PASS

Example: margin alert

margin = 18%
margin > 20% ! => FAIL

Assumptions

Assumptions are marked as inputs so reviewers can see which values are guesses, not derived facts. They can be toggled or locked during review.

Example: clear inputs for a cost model

assume tax_rate = 8%
subtotal = $73.75
total = tax_rate on subtotal => $79.65

Example: highlight growth as a guess

assume growth = 6%
revenue = $100000
next year = revenue * (1 + growth) => $106000

Ranges and confidence bands

Ranges and plus-minus syntax build uncertainty into the value itself. Results can show bands or shaded charts instead of a single number.

Example: optimistic vs cautious rates

rate = 5% ± 1%
savings = $800 * rate => $32..$48

Example: budget range planning

budget = $800..$1200
savings = budget * 4% => $32..$48

Visualization

Results should show themselves, not hide in a sidebar. These ideas make charts, pins, and alerts part of the language.

Inline charts

Chart hints tell SmartPad which chart to render for an expression or series, and where to render it (inline or pinned). The output is visual, not just a number.

Example: revenue line chart

month = 1..12
revenue = 15000 * 1.03^month
revenue => chart line

Example: category mix bar chart

food = 340
travel = 210
tools = 90
spend mix = [food, travel, tools]
spend mix => chart bar

Pinned results

Pin results to keep key outputs visible while you scroll. Pinned items can include sparklines, badges, and alerts when they drift.

Example: keep a few key KPIs visible

base plan = $45
line access = $20
fees = $8.75
monthly total = base plan + line access + fees => $73.75 pin
revenue = $420
profit = $75.6
margin = profit / revenue => 18% pin

Example: pin + alert when it drifts

cash = $45000
burn per month = $10500
cash runway = cash / burn per month => 4.3 months pin
cash runway > 3 ! => PASS

Data and Structures

Collections let SmartPad scale from a few lines to real models. You keep the narrative flow, but the UI can summarize and organize the data for you.

Lists

Arrays make aggregation patterns obvious. You can build charts or summaries without leaving the editor.

Example: simple list math

expenses = [12, 18, 9]
sum(expenses) => 39
avg(expenses) => 13

Example: list to chart

weekly steps = [6200, 7000, 5400, 8100]
weekly steps => chart bar

Inline tables

Key-value records keep related values together. They can render as cards in the variable panel while still supporting dot access.

Example: plan definition

plan = { base: $45, line: $10 }
plan.base => $45

Example: trip snapshot

trip = { flight: $420, hotel: $560 }
trip total = trip.flight + trip.hotel => $980

Data connectors

Import external data in a readable way. SmartPad would show provenance, refresh schedules, and allow cached offline reads.

Example: FX rates

fx = api("rates.usd.eur")
price eur = $120 * fx => €110.4

Example: live usage data

sessions = api("analytics.sessions.last30")
sessions => chart line

Syntax Polish

These are small, high-impact additions that make day-to-day use smoother without changing the model underneath.

Unit shorthand

Allow unit expressions without spaces for speed and natural typing. You still get automatic conversions when you ask for a different unit.

Example: compact unit entry

speed = 10m/s
speed => 10 m/s

Example: easy conversions

distance = 5km
distance to mi => 3.11 mi

Currency codes and conversions

Support currency codes as literals and enable quick conversions. The output retains currency context and uses the current FX source.

Example: code literals

100 USD => $100
EUR 200 => €200

Example: conversions

$120 to EUR => €110.4
EUR 50 to USD => $54.5

Case-insensitive keywords

Make keywords forgiving. This removes friction when users type quickly or paste text from elsewhere.

Example: percentage shortcuts still work

20% OFF $80 => $64
10% On 155 => 170.5

Example: unit keywords stay forgiving

5 m TO ft => 16.4 ft

Inline labels and tags

Attach human labels to lines. These tags can drive grouping in the variable panel, search filters, or change highlights in the editor.

Example: tag a fixed cost

rent = 1250 // fixed
rent => 1250

Example: mark a seasonal assumption

ads budget = $600 // holiday push
ads budget => $600

Human-readable dates

Treat dates as first-class values so you can compute deltas or reminders without a separate system.

Example: countdown to a deadline

launch date = 2025-03-01
today = 2025-02-17
days until launch = launch date - today => 12 days

Example: schedule from a date

start date = 2025-03-01
review date = start date + 14 days => 2025-03-15

Multi-line expressions

For long formulas, use a block to keep the document readable while still producing a single result.

Example: readable totals

base plan = $45
line access = $20
data overage fee = $8.75
total = `
  base plan
  + line access
  + data overage fee
`
total => $73.75

Example: travel cost breakdown

flight = $360
hotel = $480
food budget = $140
trip total = `
  flight
  + hotel
  + food budget
`
trip total => $980
